📖 Теория

Основы алгоритмов шифрования с открытым ключом

🔐 Асимметричное шифрование

Асимметричное шифрование (шифрование с открытым ключом) — криптографическая система, использующая пару математически связанных ключей: открытый (public) и закрытый (private). Открытый ключ доступен всем и используется для шифрования, а закрытый хранится в секрете и служит для расшифрования. Безопасность основана на вычислительной сложности факторизации больших чисел.

🌐 Открытый ключ (Public Key)

Открытый ключ — ключ, который можно свободно распространять. Он используется для шифрования данных (отправитель шифрует сообщение открытым ключом получателя) и для проверки цифровой подписи. Безопасность системы основана на невозможности вычислить закрытый ключ из открытого за разумное время.

🔑 Закрытый ключ (Private Key)

Закрытый ключ — секретный ключ, который нельзя передавать третьим лицам. Используется для расшифрования данных, зашифрованных соответствующим открытым ключом, и для создания цифровой подписи. Компрометация закрытого ключа полностью нарушает безопасность системы.

🔒 Алгоритм RSA-OAEP

RSA-OAEP (Optimal Asymmetric Encryption Padding) — схема шифрования с открытым ключом на основе алгоритма RSA с дополнением OAEP. Обеспечивает защиту от атак с выбранным шифротекстом (CCA2). В данном модуле используется RSA-OAEP с длиной ключа 2048 бит и хеш-функцией SHA-256.

✍️ Алгоритм RSA-PSS

RSA-PSS (Probabilistic Signature Scheme) — вероятностная схема цифровой подписи на основе RSA. Обеспечивает доказуемую безопасность в модели случайного оракула. В данном модуле используется RSA-PSS с длиной ключа 2048 бит, хеш-функцией SHA-256 и длиной соли 32 байта.

🌍 Практическое применение

Алгоритмы шифрования с открытым ключом применяются в: протоколе HTTPS (TLS/SSL — защита веб-трафика), инфраструктуре открытых ключей (PKI), электронной цифровой подписи документов, защищённой электронной почте (S/MIME, PGP), блокчейне и криптовалютах, VPN-соединениях.
